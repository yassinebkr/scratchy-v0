Okay so — OpenClaw just dropped version 2026.2.15, and honestly? There's some really nice stuff in here.

First up — plugin hot-reload. You can now send a SIGUSR1 signal and your plugins restart with zero downtime. No more bouncing the whole system just to swap out a module. That alone is huge.

Auth got a major upgrade too. There's a proper profile store now with OAuth credential rotation, PKCE flow support, and — this is the part I love — automatic token refresh. So you're not scrambling when tokens expire mid-session anymore.

On the memory side, compaction switched to a cache-TTL mode. You can configure staleness windows, which gives you way more control. And context pruning? They're taking million-token conversations down to about 200k — while keeping semantic coherence intact. That's... genuinely impressive.

Now — heads up — there is a breaking change. The plugin SDK import path moved. But don't panic. Just run openclaw doctor --fix and it handles the migration for you.

Performance-wise, cold starts are 40% faster thanks to lazy provider initialization. And WebSocket reconnection finally uses exponential backoff with jitter — one-second base, fifteen-second cap, twenty max attempts. Much more resilient.

Solid release overall.

💡 Verdict

Speed vs Depth, consistently. Gemini is ~2x faster for quick tasks (UI: 18s vs 27s, narration: 9s vs 10s) but ~4x slower for code (~113s vs ~30s). Both models wrote solid RetryQueue implementations with 3-state circuit breakers and typed events. Gemini's is more compact (364 lines vs 459) with UUID task IDs and a jitter factor; Opus's is larger with pause/resume and more granular lifecycle. The key difference is time-to-delivery: Gemini spends ~90s thinking before writing, Opus writes immediately. For quick creative tasks, Gemini wins. For code, Opus delivers 4x faster with comparable quality. Both at $0 (subscription tiers).
